# Claude 知识库

这是一个本地知识管理系统，用于整理和记录 Claude AI 助手的输出内容。

## 📁 目录结构

```
claude-knowledge/
├── inbox/                # 📥 临时收集区 - 快速捕获的内容待整理
├── claude-outputs/       # 📄 Claude原始输出 - 按类型分类
│   ├── 代码片段/            # 代码和技术实现
│   ├── 理论概念/            # 理论和概念解释
│   └── 工具库/              # 工具、库和框架
├── knowledge-base/       # 📚 精整理知识库 - 深度学习资料
│   ├── Python/              # Python相关
│   ├── 前端/                # 前端开发
│   ├── 架构设计/            # 系统架构
│   └── 最佳实践/            # 最佳实践集合
├── projects/             # 🚀 项目笔记 - 进行中的项目
├── references/           # 🔗 参考资源 - 外部链接和参考
├── _templates/              # 📋 笔记模板
└── _media/                  # 🖼️ 媒体文件
```

## 🏷️ 标签体系

### 一级标签（内容类型）
- `#claude-output` - Claude直接输出
- `#代码` - 代码片段
- `#概念` - 理论概念
- `#工具` - 工具和库
- `#文档` - 文档和指南
- `#问题解答` - Q&A问题解答

### 二级标签（领域/语言）
- `#Python` `#JavaScript` `#Go` `#Rust` 等编程语言
- `#前端` `#后端` `#DevOps` `#数据库` 等技术领域
- `#AI` `#架构` `#性能优化` 等专题

### 三级标签（状态）
- `#待整理` - 新导入的内容
- `#已整理` - 已分类处理
- `#精选` - 高价值内容
- `#待验证` - 需要测试验证

## 🔧 工作流

### 快速捕获流程
```
1. 复制 Claude 输出
2. 运行脚本：python claude-to-obsidian.py
3. 脚本自动：
   - 解析内容
   - 生成元数据（日期、标签、分类）
   - 保存到合适的文件夹
   - 触发 Git 自动提交
```

### 知识整理流程
```
1. 内容进入 inbox
2. 在 Obsidian 中审查和标记
3. 移动到 claude-outputs 或 knowledge-base
4. 添加双向链接关联相关内容
5. 定期查看 Graph 视图，发现知识关联
```

## 📌 最佳实践

### 笔记命名
- 使用清晰的标题：`[概念名称] - [来源/日期]`
- 例如：`Python装饰器详解 - Claude2026-04-07`

### 双向链接
- 在相关笔记中使用 `[[笔记名]]` 创建链接
- 为同类知识建立中心页面（如 `Python.md`）
- 定期查看 Graph 视图检查关联

### 代码块
- 使用语言标记：` ```python ... ``` `
- 为复杂代码添加注释说明

### Git 提交
- 系统自动每30分钟提交一次
- 手动提交时使用规范消息：
  - `feat: 添加新的知识片段`
  - `update: 更新笔记内容`
  - `fix: 修复错误`

## 🚀 快速开始

### 1. 在 Obsidian 中打开
- 打开 Obsidian
- 选择 "Open folder as vault"
- 选择 `~/Notes/claude-knowledge`

### 2. 配置 Git 同步
- 安装 "Obsidian Git" 插件
- 配置 GitHub 远程仓库
- 设置自动提交间隔为 30 分钟

### 3. 安装脚本工具
```bash
pip install pyperclip
python claude-to-obsidian.py
```
